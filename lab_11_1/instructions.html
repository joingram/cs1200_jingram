Lab 11.1

In this lab we will continue practicing how to unit test code. This time we will be writing unit tests for a class. The code you will be testing is already provided to you. Your task is simply to write tests. Follow instructions provided in this file and complete your work in Geany IDE.

Learning Outcomes

In this lab you will learn the following:

  1. Review how to write unit tests
  2. Learn how to write unit tests for a class
  3. Identify the functionality that should be tested
  4. Use setUp method

Evaluation

20 pts.Submitted all required files
20 pts.Unit tests execute without error
60 pts.Program meets requirements

Submission

Make sure you submit all assignment files.

As before, submit your work on D2L or using git. Earn 5 pts. extra credit by submitting using git by due date.

Assignment Instructions

You will need two files to complete this assignment:

1. hotdog_stand.py - this is the class you will be testing. You should not need to edit this file at all.
2. hotdog_stand_test.py - you will need to create and implement this file.

1. Review hotdog_stand.py

First review this file and the class that it contains. Try to understand what the class does and how to use it. The docstrings should be useful for understanding the functionality.

2. Create hotdog_stand_test.py

3. In hotdog_stand_test, write unit tests to verify that hotdog_stand works correctly.

  • Remember to import all necessary modules
  • Test each method in hotdog_stand
  • For each method, consider what is the intended behavior and how to verify that the behavior is working correctly.
  • Some methods are more important than others. Can you identify the difference?
  • Write more unit tests for the methods that you think are more important. For simple methods, 1 unit test should be enough.
  • Write at least 7 unit tests in total.

4. Run your unit tests

When you have at least 7 passing tests, and no failing tests or errors, you are done with this lab.
